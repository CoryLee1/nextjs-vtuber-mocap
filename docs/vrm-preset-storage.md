# VRM 预设模型存储建议

Character 面板中的「模型」列表来自 `/api/s3/resources?type=models`，当前从 **S3** 的 `vrm/` 前缀下列出 `.vrm` 文件。

## 存哪里更便宜、能放较多预设？

| 方案 | 适用场景 | 成本特点 |
|------|----------|----------|
| **当前 S3** | 已有 AWS、流量不大 | 按存储 + 请求计费，出口流量有成本 |
| **Cloudflare R2** | 需要大量预设、省流量费 | S3 兼容 API，**无出口费**，存储便宜，适合多文件 |
| **Vercel Blob** | 与 Next.js 同栈、小规模 | 免费额度后按量付费，集成简单 |
| **Git LFS + GitHub** | 开源预设包、版本管理 | 公开仓库免费，适合发布预设包 |

**推荐**：若预设数量多、访问量会起来，优先考虑 **Cloudflare R2**（S3 兼容，改 `env` 和 S3 client 配置即可）；现有 S3 继续用也可，把预设统一放到 `vrm/` 下即可在列表中展示。
